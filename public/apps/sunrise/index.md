# SUNRISE - 品質結果分析システム 開発報告

> **ビジョン：製造ラインの品質データを一元管理し、不良の傾向をいち早く掴んで品質ゼロ不良を目指す**

---

## 1. 現在の課題（Problem）

三田工場の品質管理業務で日常的に発生している課題を整理しました。

### 1-1. 検査結果の確認が属人的・事後的

各ラインの品質検査結果は担当者が個別に確認しており、全体の傾向を把握するには時間がかかる。不良が増加していても「数が増えてから気づく」という後手の対応が常態化している。

### 1-2. データが散在していて比較・分析しにくい

検査結果データが複数の場所に分散しており、ライン間・期間間の比較が困難。不良要因を特定するための横断分析ができていない。

### 1-3. トレンドの変化を見逃しやすい

日次・週次の品質推移を可視化する手段がなく、「じわじわ悪化している」変化を見逃しやすい。問題が顕在化してから対策を打つため、流出リスクと手戻りコストが発生する。

```
現状の品質管理フロー:
├── 検査結果を各担当者が個別に確認    ← 全体像が把握できない
├── 問題が大きくなってから報告        ← 早期発見できない
├── 原因調査（人力・経験頼り）        ← 時間がかかる
└── → 「不良が出てから直す」後手の品質管理が続く
```

---

## 2. SUNRISEでこう変わる（Solution）

| # | 現状（BEFORE） | 導入後（AFTER） |
|---|---|---|
| 1 | 検査結果を各自がバラバラに確認 | **Webブラウザで誰でも最新状況を確認**できる |
| 2 | 全体トレンドの把握に時間がかかる | **ダッシュボードで一目**に品質推移を把握 |
| 3 | ライン間の比較が困難 | **ライン別・日別の比較**がグラフで即確認 |
| 4 | 不良要因の分析が属人的 | **データドリブンな要因分析**で客観的判断 |
| 5 | 過去データの参照が手間 | **期間指定で履歴を即検索**・表示 |

### SUNRISEの処理構成

```
SUNRISE
└── データ収集
    ├── 品質検査システムから結果を取得
    └── データベースへ蓄積・整理
        │
        ├── 分析・集計エンジン
        │   ├── ライン別品質集計
        │   ├── 不良種別集計
        │   ├── 時系列トレンド分析
        │   └── 良品率・不良率算出
        │
        └── Webアプリ（ダッシュボード）
            ├── ホーム：全体サマリー
            ├── ライン別詳細ビュー
            ├── 期間比較グラフ
            └── 不良要因ドリルダウン
```

## 4. 主要機能の詳細

### 4-1. ホームダッシュボード

全ラインの品質状況を一画面で把握できるサマリービュー。良品率・不良率・検査数を日別に集計し、異常傾向を一目で確認できる。

```
ホーム画面
  ├── 本日の検査サマリー（全ライン合計）
  ├── ライン別 良品率ランキング
  ├── 直近7日間のトレンドグラフ
  └── 要注意ライン（閾値を下回ったライン）のアラート表示
```

### 4-2. ライン別詳細分析

ライン単位でのドリルダウン分析。不良種別の内訳・時間帯別の発生傾向・過去比較などを表示し、「どの工程で・どんな不良が・いつ多いか」を把握できる。

```
ライン詳細ビュー
  ├── 不良種別 円グラフ（パレート分析）
  ├── 時間帯別 発生分布
  ├── 日別推移グラフ（良品数 / 不良数）
  └── 前週・前月との比較表示
```

### 4-3. 期間指定・履歴検索

任意の期間を指定して過去データを即時参照。月次レポート作成や、不具合発生時の原因調査に活用できる。

## 5. 実装済みの価値

| 価値 | 詳細 |
|---|---|
| **品質の見える化** | 全ラインの検査結果をWebブラウザからいつでも確認可能 |
| **早期異常検知** | トレンドグラフで「じわじわ悪化」をいち早く発見 |
| **属人化の解消** | 誰でも同じ情報にアクセスでき、担当者依存を脱却 |
| **データドリブン** | 経験・勘ではなくデータに基づく品質改善判断が可能 |
| **社内即時アクセス** | 社内ネットワークからブラウザのみで利用可（インストール不要） |

## まとめ

1. 品質検査結果を**全ラインで一元管理**し、「誰でもいつでも最新状況を確認できる」体制を構築
2. トレンドグラフ・ライン比較により、「じわじわ悪化する品質の変化」を**データで早期に発見**
3. その先に**FOURESIGHTとの連携・AI予兆検知**との統合を通じ、品質不良ゼロの現場実現を目指す

> **SUNRISE**
> データが語る、品質の今を見逃さない。
