# hint webアプリ - 製造現場デジタル化ツール群 調査報告

> **ビジョン：製造現場のあらゆる情報をスマホ1台で即確認・即入力し、紙・Excelゼロの現場を実現する**

---

## 1. 現在の課題（Problem）

姫電製造現場で日常的に発生している情報収集・実績入力の課題を整理しました。

### 1-1. 生産状況の確認に現場へ足を運ぶ必要がある

各ラインの生産台数・稼働状況を把握するには、管理者が現場に直接出向くか、Excelのラインモニタを開く必要がある。Excel版ラインモニタは複数ファイルを同時に開くと動作が不安定になる問題もあった。朝礼・巡回のたびに情報収集に時間がかかり、異常への対応が遅れる。

### 1-2. 設備のない手作業工程の実績がhintに上がらない

目視検査・手直し工程など、設備を持たないマニュアル工程ではhintへの実績登録手段がなかった。結果として**工程飛び（品質不具合）**が発生するリスクがあっても検知できない状態だった。

### 1-3. ワークのトレーサビリティ確認が煩雑

不良が発生した際、そのワークがどの工程でNGになったか、どのロットの部品を使っているかを調べるには、hintの管理画面を操作できる担当者に依頼する必要があった。現場や朝市で即座に確認できる手段がなかった。

```
現状の業務フロー:
├── 生産状況確認  ← 現場へ足を運ぶ or Excel（動作不安定）
├── 手作業工程    ← hint実績なし → 工程飛びリスク
├── 鮮度管理      ← 子部品の制限時間を手動で管理
├── NG→OK修正    ← hint管理者に依頼、修正履歴が残りにくい
└── → 情報がデジタルに繋がらず、現場判断が遅れる
```

### 1-4. 子部品の鮮度（制限時間）管理が手作業

組付け工程で使用する子部品には制限時間（鮮度）があるが、管理は人の記憶や紙メモに依存していた。鮮度切れ部品の混入リスクを排除できていなかった。

## 2. hint webアプリでこう変わる（Solution）

| # | 現状（BEFORE） | 導入後（AFTER） |
|---|---|---|
| 1 | 生産状況確認のため現場へ移動 | **スマホ・PC**でどこからでもリアルタイム確認 |
| 2 | Excel版ラインモニタが複数起動で不安定 | **Webアプリ**でExcel無関係、動作安定 |
| 3 | 設備なし工程はhintに実績が上がらない | **これ完成！/これ完成２**でスマホから即入力 |
| 4 | ワーク素性の調査に専任担当者が必要 | **これ何？**でシリアル入力だけで誰でも即検索 |
| 5 | NG→OK修正の履歴管理が不十分 | **NG→OK修正アプリ**で理由記録・修正履歴を自動保存 |
| 6 | 子部品の鮮度管理が人の記憶頼み | **鮮度OK？**で制限時間を自動チェック・エラー通知 |

### hint webアプリ構成

```
hint webアプリ（8種）
├── 生産モニタ系
│   ├── 今何台？        ← ライン別生産台数をリアルタイム確認
│   ├── ラインモニタ    ← 全工程のOK/NG・トラブルを一覧表示
│   └── はかどってる？  ← 生産数と計画数の差をグラフ表示（今何台？上位版）
├── トレーサビリティ系
│   └── これ何？        ← シリアルでワーク素性を即検索
├── 実績入力系
│   ├── これ完成！      ← 設備なし工程のOK/NG入力＋工程飛びチェック
│   └── これ完成２      ← バーコードでセッティング情報設定＋ポカヨケ入力
└── 品質管理系
    ├── NG→OK修正      ← 手直し後のNG→OK書き換え・修正履歴保存
    └── 鮮度OK？        ← 子部品の制限時間チェック
```

hintデータベースに蓄積された製造実績を軸に、確認・入力・修正・検索のすべてをスマホ1台で完結できます。

## 4. 主要アプリの詳細

### 4-1. 今何台？（生産台数リアルタイム確認）

いつでもどこでもアプリを開けば、見たいラインの現在の生産台数を一覧で確認できる。稼働状況を色で把握でき、過去日付の指定表示も可能。PC版・スマホ版の両方に対応。

```
入力: ラインを選択（任意に複数選択可能）
  ↓
表示: 生産台数 / 現時点での計画数 / 進捗グラフ
  ↓
判断: 色で稼働状況を即把握（稼働中・停止・未生産）
```

### 4-2. これ何？（ワーク素性検索）

シリアルを入力するだけでワークの素性が分かる。NG発生工程・流動工程・部品ロット・測定値など様々なデータを把握可能。Bluetooth接続したバーコードリーダーで現場でのシリアル読み込みにも対応。

```
入力: シリアル番号（手入力 or バーコードリーダー）
  ↓
検索: NG検索 / 工程検索 / ロット検索 / 測定値検索
  ↓
出力: NG発生工程、流動工程、使用部品ロット、各工程の測定値
```

### 4-3. これ完成！（手作業工程の実績入力）

目視チェック工程や手直し工程など、シーケンスがない設備でもOK/NG情報を簡単にhintに登録できる。工程飛びチェック機能により、品質不具合である「工程飛び」を防止できる。

```
入力: 工程コード・機種名 → シリアル → OK/NG選択
  ↓
処理: hintへ実績登録 ＋ 工程飛びチェック（前工程完了確認）
  ↓
出力: 登録完了 or 工程飛びエラー通知
```

### 4-4. これ完成２（バーコード＋ポカヨケ実績入力）

QRコードを読み込むことでセッティング情報（拠点・ライン・工程など13項目）を一括設定可能。入力ミスのポカヨケを実装し、データの信頼性を向上。

```
入力: QRコード読み取り → セッティング情報13項目を自動設定
  ↓
入力: シリアル → NG理由（選択）→ OK/NGボタン
  ↓
出力: hintへ実績登録（紐づけデータ含む）
```

### 4-5. NG→OK修正

手直しや品管判断でOKとなったワークのhintデータを書き換えるアプリ。修正理由（カメラ誤判定・承認・データ間違い・その他）を選択して記録し、修正履歴は必ず残る。ユーザー管理マスタで使用者を制限可能。

### 4-6. 鮮度OK？

子部品ごとに制限時間（鮮度）を設定し、シリアルを入力・実行するだけで鮮度チェックが可能。鮮度NGの場合はエラーメッセージを出力し、制限時間切れ部品の混入を防止する。

### 4-7. ラインモニタ

全工程の生産稼働状況・OK/NG数・トラブル数をリアルタイム表示。工程名が赤色表示されればトラブル停止中。トラブル・NGをクリックすると詳細が表示される。Excel版と異なりWebアプリのため複数起動しても動作が安定している。

### 4-8. はかどってる？

「今何台？」の上位版。生産数と計画数の差をグラフでリアルタイム表示し、日ごと・月ごとの表示に対応。大型モニタ向けに解像度を向上させ、PPH表示・目標値グラフ描画設定機能を追加。

## 6. 実装済みの価値

| 価値 | 詳細 |
|---|---|
| **どこでも即確認** | 生産台数・稼働状況をスマホから即座に確認。現場へ足を運ぶ移動コストをゼロに |

| **工程飛び防止** | 「これ完成！」の工程飛びチェック機能で品質不具合を未然にブロック |
| **誰でもトレーサビリティ** | シリアル1つでNG原因・流動履歴・部品ロットを誰でも即検索可能 |
| **鮮度管理の自動化** | 子部品の制限時間をシステムが自動判定。人の記憶・紙管理から脱却 |
| **修正履歴の完全記録** | NG→OK修正の理由・実施者・日時が必ず記録され、品質証跡を確保 |
| **Excel依存からの脱却** | ラインモニタをWebアプリ化し、Excelの複数起動による動作不安定問題を解消 |
